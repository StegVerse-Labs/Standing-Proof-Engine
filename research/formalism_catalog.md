# StegVerse Mathematical Formalism Catalog

Status: draft formalism catalog
Placement: `research/`
Standing: PARTIAL until machine-readable schemas and executable transition-cell tests are added.

## 1. Assumptions

This catalog defines the research vocabulary for mapping mathematical problems into StegVerse Formalisms. It does not claim that the vocabulary solves any listed problem.

A formalism is treated as a transition system with standing-aware artifacts:

```text
object state
→ local transition
→ admissibility predicate
→ invariant or obstruction check
→ global consequence question
→ standing result
```

## 2. Core objects

| Object | Meaning |
|---|---|
| `State` | A mathematical configuration at a declared stage. |
| `Transition` | A local transformation from one state to another. |
| `Cell` | A transition-periodic-table classification entry. |
| `Invariant` | A quantity, relation, obstruction, or structure preserved or monotonically controlled by transitions. |
| `Boundary` | A condition that separates admissible from inadmissible transitions, finite from divergent behavior, smooth from singular behavior, or local from global standing. |
| `Orbit` | A transition path generated by repeated application of local rules. |
| `Receipt` | A reproducible record of a transition, invariant check, or standing decision. |
| `Standing` | The declared proof status of an artifact: `PASS`, `PARTIAL`, or `FAIL`. |

## 3. Transition schema

Each transition should be expressible as:

```text
T: State(domain, context, constraints) -> State(domain, context, constraints)
```

Required fields:

```text
transition_id
input_state
output_state
local_rule
admissibility_predicate
invariant_claim
failure_mode
standing_status
```

## 4. Transition periodic table cell schema

Each transition cell should eventually declare:

```text
cell_id
name
symmetry_signature
conservation_signature
reversibility_signature
scale_signature
periodicity_signature
obstruction_signature
complexity_signature
example_problem_refs
```

## 5. Standing levels for mathematical formalism artifacts

### PASS

The artifact is independently reconstructable and verifies exactly the declared claim.

### PARTIAL

The artifact gives a useful mapping, schema, or conjectural direction, but at least one reconstruction, proof, citation, or executable test gap remains.

### FAIL

The artifact is missing, malformed, inconsistent with known facts, or claims more than it proves.

## 6. Problem encoding template

Each problem encoding must include:

1. problem statement;
2. source-standing note;
3. StegVerse mapping;
4. state definition;
5. transition rule;
6. candidate transition table cells;
7. invariants or obstruction candidates;
8. known constraints and barriers;
9. executable or reviewer reconstruction plan;
10. non-claims;
11. standing status.

## 7. Calibration standard

A calibration problem is useful when it is simple enough to encode precisely but deep enough to expose whether the formalism adds insight beyond renaming the problem.

The first calibration set is:

- Collatz conjecture;
- Jacobian conjecture;
- Caccetta-Haggkvist conjecture.

## 8. Non-claims

This catalog does not claim to solve any open problem.

This catalog does not claim that a transition-table cell is a proof.

This catalog does not claim that `PASS` for repo structure equals `PASS` for a mathematical theorem.
